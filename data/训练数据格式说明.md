# 训练数据格式说明

本目录下每个子目录（如 `1/`）为一组训练样本，需包含 **同名的 .npz** 与 **同名的 .mp3**，例如：

- `1/1.npz` — 打包的帧参数与词-帧对应表
- `1/1.mp3` — 整段音频（与 npz 内时间戳对应）

训练时由 `WordFrameDataset` 使用：指定 `packed_npz_path=Project/data/1/1.npz`、`audio_path=Project/data/1/1.mp3` 即可。

---

## 1. `*.npz` 格式（打包数据）

由 `demo/project/data/pack_result_to_npz.py` 从「每帧一目录 + word_frames.csv」打包得到。  
`np.load(..., allow_pickle=True)` 后为字典，包含：

| 键 | 形状/类型 | 说明 |
|----|-----------|------|
| `shape` | (300,) float32 | 首帧人脸 shape，全序列共用 |
| `exp` | (N, 100) float32 | 每帧表情系数（FLAME 100 维） |
| `jaw` | (N, 3) float32 | 每帧下颌角 |
| `global_pose` | (N, 3) float32 | 每帧全局姿态 |
| `frame_names` | (N,) object | 帧目录名列表，与 exp/jaw/global_pose 行一一对应 |
| `init_shape` | (300,) float32 | 与 `shape` 相同，用于 init |
| `init_exp` | (100,) float32 | 首帧 exp |
| `init_jaw` | (3,) float32 | 首帧 jaw |
| `init_global_pose` | (3,) float32 | 首帧 global_pose |
| `word_frames_csv` | (1,) object | 字符串：整份 word_frames.csv 的文本内容 |

- **N** = 总帧数（与 `frame_names` 长度一致）。  
- 若为分块 npz（`*_part0.npz` 等），还会多一个 `frame_offset`（int），表示本块在全局中的起始帧下标。

### `word_frames_csv` 内容（CSV 表）

表头一行，列名：`word`, `start_sec`, `end_sec`, `start_frame`, `end_frame`, `frame_folders`。

| 列 | 说明 |
|----|------|
| word | 词文本 |
| start_sec / end_sec | 该词在音频中的起止时间（秒） |
| start_frame / end_frame | 起止帧号（与 N 帧对应） |
| frame_folders | 该词对应的帧目录名，用 `\|` 分隔，需与 `frame_names` 中的名称一致 |

示例一行：

```text
word,start_sec,end_sec,start_frame,end_frame,frame_folders
hello,0.12,0.48,4,14,000004_000|000005_000|...|000014_000
```

训练时按行取「词 + 时间 + 帧目录」：用 `start_sec`/`end_sec` 在音频中截段，用 `frame_folders` 在 npz 的 exp/jaw/global_pose 中取 GT (T, 106)。

---

## 2. `*.mp3` 格式（音频）

- 与 npz **同一目录、同名**（仅扩展名不同），如 `1.mp3` 配 `1.npz`。
- 整段音频，采样率无硬性要求；加载时由 `librosa` 重采样到 **16000 Hz**（与 Wav2Vec2 / 配置中的 `AUDIO_SAMPLE_RATE` 一致）。
- CSV 中的 `start_sec` / `end_sec` 必须落在此音频长度内。

## 4. 如何生成 npz + mp3（获取训练数据）

使用仓库内一条龙脚本，从**视频**直接生成 `out_dir/<视频名>.npz` 与 `out_dir/<视频名>.mp3`（不写大量中间帧目录）。输出与视频同名，例如 `2.mp4` → `2.npz`、`2.mp3`；若视频同目录已有同名 .mp3 则直接使用，不复制。

- **脚本**：`Project/scripts/video_to_training_data.py`
- **运行**：在**仓库根**执行。若出现 `No module named 'inferno.datasets'`，说明当前环境未把本仓库当包使用，可二选一：
  1. **推荐** 在仓库根安装一次：`pip install -e .`（可编辑安装，之后可直接 `python Project/scripts/...`）；
  2. 或带环境变量运行：`PYTHONPATH=. python Project/scripts/video_to_training_data.py ...`（不安装）。

示例：

```bash
# 输出与视频同名：2.mp4 → Project/data/2/2.npz 与 2.mp3；若已有 2.mp3 则直接使用
python Project/scripts/video_to_training_data.py --video Project/data/2/2.mp4 --out_dir Project/data/2

# 指定单独音频（会写入 out_dir/xxx.mp3）
python Project/scripts/video_to_training_data.py --video 1.mp4 --audio 1.wav --out_dir Project/data/1
```

常用参数：

| 参数 | 说明 |
|------|------|
| `--video` | 输入视频路径（必填） |
| `--audio` | 可选；省略则先查视频同目录同名 .mp3，无则从视频抽出并保存为 `out_dir/<视频名>.mp3` |
| `--out_dir` | 输出目录，将写入 `out_dir/<视频名>.npz` 和 `out_dir/<视频名>.mp3` |
| `--path_to_models` | FaceReconstruction 模型目录，默认仓库 assets |
| `--model_name` | 模型名，默认 `EMICA-CVT_flame2020_notexture` |
| `--fps` | 词→帧对齐用帧率（默认 30）；若能从视频元数据解析则自动覆盖 |
| `--use_cpu` | 全部使用 CPU |
| `--whisper_cpu` | 仅 Whisper 用 CPU（人脸仍用 GPU；遇 cuDNN 报错时加此参数） |
| `--use_modelscope` | Whisper 从 ModelScope 下载（国内推荐） |
| `--whisper_model` | Whisper 模型：base / small / medium 或本地路径 |
| `--whisper_cache` | Whisper 模型目录，默认 `Project/model/whisper_models`（可事先放入或通过 --use_modelscope 下载） |

脚本会先解帧并做人脸检测到临时目录，跑 FaceReconstruction 在内存中收集每帧 codes，再对音频做 Whisper 词对齐、拼 `word_frames_csv`，最后写出与视频同名的 npz 与 mp3，并删除临时目录。

## 5. 与 FLAME/口型维度的对应

- 单帧口型参数 **106 维** = exp(100) + jaw(3) + global_pose(3)。  
- npz 中的 `exp`、`jaw`、`global_pose` 与 demo 中 `init_frame0.npz`、`Result/3` 下每帧的 `exp.npy`、`jawpose.npy`、`globalpose.npy` 含义一致，可直接用于训练或导出 FLAME mesh（见 `Project/emica_export`）。
